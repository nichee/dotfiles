# /review — Scoped Code Review

Review code against requirements or a rubric. Never modify files unless explicitly asked.

## Steps

1. Read the requirements file, assignment spec, or rubric first.
2. Read all relevant source files.
3. Compare implementation against requirements point by point.
4. Output a gap list — what's missing, incorrect, or incomplete.
5. For written content (reports, docs): suggest minimal wording changes only. Never rewrite sections. Show suggestions as inline diffs, not replacements.

## Rules

- Do NOT edit any files during this skill — review only.
- Do NOT inflate scores or say something is good when it isn't.
- If something is not ready, say exactly what's wrong and what's needed to fix it.
- Wait for the user to approve before making any changes.
