Always create a persistent `.md` file (e.g. `PLAN.md`) at the start of any project or significant feature work.

**Why:** User explicitly values repeatable workflows and wants architecture/planning captured in a durable file, not just in chat. This is part of how they like to develop alongside Claude.

**How to apply:** When starting a new project or a non-trivial task, proactively create or update a `PLAN.md` in the project root before writing code. Keep it updated as the plan evolves.
